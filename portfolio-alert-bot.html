Alert Bot – Application & Infrastructure Monitoring (MyEyeOps)

  • Category: DevOps / Monitoring
  • Tools: Python, Linux, MySQL, Cron, Jenkins, SMTP, REST APIs
  • Role: DevOps Engineer
  • GitHub: Repository

Project Overview

MyEyeOps is a custom-built alerting and monitoring solution designed to continuously track application health and infrastructure metrics. The system performs automated health checks on services, logs, and system resources to enable early incident detection and proactive response.

Key Responsibilities & Implementation

  • Designed and developed a Python-based alert bot for application and infrastructure monitoring.
  • Implemented periodic health checks for services, CPU, memory, and disk utilization.
  • Stored alert thresholds, configurations, and execution status in a MySQL database.
  • Triggered automated email notifications using SMTP when thresholds were breached or services went down.
  • Automated execution using Linux cron jobs and Jenkins scheduled pipelines.

Impact

  • Reduced manual monitoring effort through automated health checks.
  • Improved incident detection time and overall system reliability.
  • Enabled proactive monitoring instead of reactive issue handling.

Future Enhancements

  • Integrate Slack and PagerDuty for real-time alerting.
  • Add Prometheus and Grafana dashboards for advanced observability.
  • Extend monitoring to Kubernetes workloads.
